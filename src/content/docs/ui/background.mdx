## Usage

Backgrounds are **opt-in** — there is no default. A chart is bare until you compose
one, and `variant` is required (there is no `"none"`).

Compose `ChartBackground` inside a **cartesian** chart — `NQLineChart`, `NQBarChart`,
`NQAreaChart`, `NQComposedChart` (and `NQSparklineChart`):

```tsx
import { NQLineChart, Line, XAxis, Legend, Tooltip } from "@nqchart/line-chart";
import { ChartBackground } from "@nqchart/background";

<NQLineChart data={data} config={chartConfig} className="h-full w-full p-4" xDataKey="month">
  <ChartBackground variant="dots" />
  <XAxis dataKey="month" />
  <Legend />
  <Tooltip />
  <Line dataKey="desktop" />
  <Line dataKey="mobile" />
</NQLineChart>
```

### Not available on pie, radar, radial, treemap, funnel, calendar

Those charts have no x/y axes, so there is no plot area to contain a pattern — they
intentionally don't export `Background`.

### Contained by the axes

The pattern is clipped to the **plot area between the x and y axis**, measured from
the chart at runtime. It stays aligned no matter the container size or how wide your
axis labels are (`1,000` vs `$1,250,000.00`), so it never bleeds under the labels.

Sparkline is the exception: it has no visible axes, so its pattern fills the plot.

### Background pattern vs chart grid

| API | What it is |
|-----|------------|
| `<ChartBackground variant="graph-paper" />` | Decorative SVG wallpaper (small repeating squares) |
| `<Grid />` | Optional **y-axis split lines** at value ticks (wide spacing) |

**Use one, not both.** `<Grid />` is a functional value guide; the background is
decoration. Stacking two pattern layers competes for the same space and makes values
harder to read — the previews below omit `<Grid />` for that reason. Add `<Grid />`
only when you want value guides *instead of* a pattern.

## Custom Variant

You can add your own background style by defining an SVG `<pattern>` and registering it in `PATTERN_MAP` inside `src/registry/ui/background.tsx`.

1. Add the new name to the `BackgroundVariant` union.
2. Create a pattern component that returns a `<pattern>` with your SVG shapes.
3. Register the pattern component in `PATTERN_MAP`.

Then use your variant:

```tsx
<ChartBackground variant="custom-pattern" />
```
