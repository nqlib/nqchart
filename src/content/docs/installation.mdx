BeeCharts ships as a **shadcn-compatible registry** — not an npm package. The shadcn CLI copies chart source into your repo (typically `components/beecharts/`). Agent **skills** are separate: they install markdown guidance for Cursor, Claude Code, etc., and do **not** copy chart files.

## Steps
<Steps>
  <Step>
    <StepTitle>Install Apache ECharts</StepTitle>
    <StepContent>
      <StepDescription>
        Install Apache ECharts and motion, the runtime dependencies for all chart components.
        
        Learn more at the <Link href="https://echarts.apache.org/en/index.html" _blank>Apache ECharts documentation</Link>.
      </StepDescription>
      <CommandBlock commands={["echarts", "motion"]} />
    </StepContent>
  </Step>
  
  <Step>
    <StepTitle>Setup shadcn/ui</StepTitle>
    <StepContent>
      <StepDescription>
        Initialize shadcn/ui in your project. This sets up Tailwind CSS, theme configuration, and `components.json`. Skip this step if you already have shadcn/ui installed.
        
        Check out the <Link href="https://ui.shadcn.com/docs/installation/next" _blank>shadcn/ui Next.js installation docs</Link> for more details.
      </StepDescription>
      <CliBlock action="init" commands={[]} />
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Add the BeeCharts registry namespace</StepTitle>
    <StepContent>
      <StepDescription>
        Register the `@beecharts` namespace in your project's `components.json` so `shadcn add @beecharts/…` resolves to this site. Skip if already configured.
      </StepDescription>
      ```json
      {
        "registries": {
          "@beecharts": "https://beecharts.vercel.app/r/{name}.json"
        }
      }
      ```
      <StepDescription className="mt-4">
        Alternative: install by full URL without the namespace — `pnpm dlx shadcn@latest add https://beecharts.vercel.app/r/bar-chart.json`.
      </StepDescription>
    </StepContent>
  </Step>
  
  <Step>
    <StepTitle>Add chart components (shadcn registry)</StepTitle>
    <StepContent>
      <StepDescription>
        Add chart source with the **shadcn CLI**. Replace `{chart-name}` with your chart type (e.g. `area-chart`, `bar-chart`, `line-chart`, `pie-chart`). Registry dependencies (`@beecharts/chart`, `@beecharts/tooltip`, etc.) install automatically.
      </StepDescription>
      <CliBlock commands={["@beecharts/{chart-name}"]} />
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Optional — Chart recipes (BI data helpers)</StepTitle>
    <StepContent>
      <StepDescription>
        For histogram, Pareto, heatmap, bullet, gauge, and box plot patterns, install the shared data helpers via the same shadcn registry. See [Chart recipes](/docs/chart-recipes) for usage.
      </StepDescription>
      <CliBlock commands={["@beecharts/chart-recipes"]} />
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Optional — Agent skill (skills CLI)</StepTitle>
    <StepContent>
      <StepDescription>
        **Not the same as chart install.** The [skills CLI](https://ui.shadcn.com/docs/skills) copies agent guidance into `.agents/skills/beecharts/` — when to pick each chart, compose children, theme colors, BI recipes, and the homepage demo pattern. It does **not** run `shadcn add` or copy React components.

        Run from your **app project root** (not inside the beecharts repo). Use `--list` to see available skills. When **contributing to this repository**, use `beecharts-dev` and `beecharts-docs` (see root `AGENTS.md`) — not the consumer skill.
      </StepDescription>
      <SkillsBlock commands={["ctesibius/beecharts", "--skill", "beecharts", "-y"]} />
      <StepDescription className="mt-4">
        Canonical skill source: `skills/consumer/beecharts/` in the repo. After install, agents load `.agents/skills/beecharts/`. HTTP: [/.well-known/agent-skills/beecharts/SKILL.md](https://beecharts.vercel.app/.well-known/agent-skills/beecharts/SKILL.md) · [llms.txt](https://beecharts.vercel.app/llms.txt)
      </StepDescription>
    </StepContent>
  </Step>
</Steps>
